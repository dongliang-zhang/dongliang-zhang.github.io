生活故事会

发表于 2019-08-16 分类于 杂(others) 评论数： 阅读次数：

锅糊了

厨房传来阵阵浓烟，焦糊的味道迅速蔓延，伴随着老妈的自责声：”遭了，遭了，忘记关火了，烧干了！”。恰好老爸从外边回来，目睹了这刺鼻的一幕，压抑不住的怒气责怪道：”做个饭还能把锅烧着了？”。

常在河边走哪有不湿鞋啊，不做永远不会错。

搓麻

一个麻将局，我买了一副扑克决定代替每次找地方换钱的尴尬和大家每局微信支付的无感，大家也很赞同，但是我的下一个方案马上被三家否定，”我们每个人一个花色，1-10代表对应的点数，J-A代表20点”, “不不不，太麻烦了，就每个人十张牌多简单”三家一致的道出同样的想法，少数服从多数，结果一局玩完就换回了我的方案。

真理本来就不是掌握在多数人的手里，有经验者或者思考更深入者才掌握更佳方案，但是往往多数人更容易达成一致，更不幸的是很多事不能那么快验证对错，或者不好评价对错。项目中你经历了多少这样的事？

锤子的进化

看图想自己的故事，讲自己的道理。

  • 进化就是这样，先要有再不断的完善。万事开头难，先要勇敢的迈出第一步。
  • 别人推荐图四的锤子，你拿着图二的锤子说：”我的锤子很好，不需要你那玩意”。结果锤子没用多久碎了。
  • 别人给你推荐图五的锤子，你拿着图四的锤子说：”你那个太复杂了，我的锤子够用了”。结果需求变了，场景要求更多了，此时撮襟见肘的你。当有了进化版，不防试试。
  • 图六，此时的你，架势很凶猛，表情到位，但是硬件跟不上啊。所谓磨刀不误砍柴工！自己琢磨琢磨吧。

多少项目让你有图六的感觉。个人认为智慧的结晶，经验的总结，先去研究实践领悟，比强调灵活应用在刚开始更重要，首先问问自己懂吗？再提灵活变通，小心中国式的聪明。

你真的了解了吗

信息的准确 - 表述方和接收方表述和理解上的差异 - 信息传递过程中的失真 = 你了解的信息。了解真正需求才是关键。

个人认为目前项目中Consultants和PO的价值貌似更高，合理的缘由在于他们是信息的源头，如果弄错了南辕北辙，劳民伤财。这也是经验的价值。如果你是舵手更应该谨慎些。

你感觉某某同事怎么样？

某天PM找我，想通过我了解一下某个同事工作怎么样，我一想领导对我这么信任，来问我说明也信任我的识人能力，所以我就公平公正的说了：”此处省去一些字，最后一句目前欠些火候，还不行”。PM马上回复道：”我问另一位某某同事的时候，他的评价很高，认为挺好的”。一口老血喷出又被我咽回去的感觉。默默心里嘀咕：”你玩领导权术也别告诉我啊，我以为你信任我评论才问我，现在弄得我好小人的感觉。如此这样，下次你再问我，我只能说非常好或者拒绝回答。”

如果你遇到这种情况知道怎么做了吧。不知道是不是领导故作聪明，难道忘记了一千个读者有一千个哈姆雷特吗？如果采用大数据的方式，也别说漏啊。

不过我更想说的是下面这层意思，天平要有所倾斜，才能鼓励真实的情况被反馈。鼓励积极或信任的人，比永远保持平衡重要。

对于架构师，在技术选型上有两种倾向：偏保守或者偏激进。偏保守的，会多选择已经经过多年使用，成熟稳定的技术，这样不确定性因素少，掉坑机率小。偏激进的，会多选择新出现的技术，因为新技术往往功能和性能上都更佳，可以更好更快的满足需求。
两种倾向各有优劣，我无意从技术层面上讨论哪种更好。但如果要打造一个有浓厚技术氛围的团队，那么最好是能将天平向激进一端倾斜一些。过于保守追求稳妥的技术团队是很难形成学习型氛围的。

感谢支持，让我安静的做蚂蚁梦！
